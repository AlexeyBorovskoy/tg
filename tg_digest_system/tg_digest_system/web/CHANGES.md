# Изменения в логике веб-интерфейса

## Упрощение логики доступа

### Что изменилось:

1. **Убрана проверка через Bot API** - система больше не проверяет доступ через бота
2. **Убран fallback на доверие клиенту** - система работает только с чатами где она присутствует
3. **Упрощена проверка доступа** - только через системную Telethon сессию

### Новое сообщение об ошибке:

Если система не имеет доступа к чату, клиент получает сообщение:
```
Система не имеет доступа к чату {chat_id}. 
В настоящее время система находится в стадии тестирования и поддерживает работу 
только с чатами, где она присутствует. Работа со сторонними ресурсами пока не поддерживается.
```

### Что осталось:

1. **Клиент может задать куда присылать дайджест** - поле `recipient_telegram_id` в форме
2. **Клиент может скачать инженерный документ** - новая кнопка "Документ" в таблице каналов
3. **Упрощённый интерфейс** - убраны предупреждения о необходимости добавления бота

## Новый endpoint

### GET `/api/channels/{channel_id}/document`

Возвращает сводный инженерный документ канала в формате Markdown.

**Параметры:**
- `channel_id` - ID канала
- `user_telegram_id` - Telegram ID пользователя (query параметр)

**Ответ:**
- Файл Markdown с инженерным документом
- Или 404 если документ не создан/не найден

## Изменения в UI

1. **Главная страница:**
   - Обновлено предупреждение о тестовой версии
   - Убраны упоминания о добавлении бота

2. **Страница каналов:**
   - Убрана колонка "Доступ" (всегда "✓ Доступен")
   - Добавлена кнопка "Документ" для скачивания инженерного документа
   - Убраны предупреждения о необходимости настройки

## Миграция БД

Поля `bot_required` и `setup_instructions` больше не используются, но остаются в таблице для обратной совместимости.
